# Спецификация библиотеки «UniConv»

---

**Раздел 1. Общая информация**

| Поле | Значение |
|------|----------|
| Название библиотеки | UniConv+ (Universal Unit Converter Plus) |
| Назначение | Расширяемая библиотека конвертации единиц измерения с поддержкой UI-форматирования и инженерных таблиц |
| Язык реализации | Python 3.x |
| Соответствие стандартам | ГОСТ 8.417-2002 (ГСИ) |
| Версия спецификации | 2.0 |

---

**Раздел 2. Архитектура и Особенности**

> **Цель:**
> Объединить гибкость динамического добавления единиц (как в `uniconv`) с жесткой инженерной логикой и полным набором данных `UnitManager`.
>
> **Ключевые изменения относительно версии 1.0:**
> 1.  **Двуязычный API:** Обращение к параметрам возможно как по русскому названию (`Давление`), так и по английскому ключу (`pressure`).
> 2.  **Разделение Код/UI:** У каждой единицы есть ASCII-код для использования в коде/БД (напр. `м3/ч`) и UI-представление для вывода на экран (напр. `м³/ч`).
> 3.  **Табличные конвертации:** Поддержка нелинейных зависимостей (Твердость) через встроенные таблицы данных.
> 4.  **Расширенная термодинамика:** Добавлена шкала Реомюра.

---

**Раздел 3. Функциональные требования (API)**

**3.1. Основные методы класса `UnitConverter`**

| Метод | Сигнатура | Описание |
|-------|-----------|----------|
| **convert** | `(val, from_u, to_u, type) -> float` | Сквозная конвертация. Аргумент `type` принимает как Ru ("Давление"), так и En ("pressure") ключи. |
| **get_available_units** | `(type) -> list[dict]` | Возвращает список словарей: `{'code': 'м3/ч', 'ui': 'м³/ч', 'name': 'Кубический метр в час'}`. |
| **get_parameter_info** | `(type) -> dict` | Возвращает метаданные параметра (Ru name, En name, System unit code/ui). |
| **add_unit** | `(...)` | Динамическое добавление пользовательских единиц (наследие `uniconv`). |

---

**Раздел 4. Бизнес-правила (Логика)**

| ID | Правило | Формула / Логика |
|----|---------|-------------------|
| **BR-01** | **Линейная конвертация** | $$Val_{sys} = Val_{in} \times Factor$$ |
| **BR-02** | **Температурные сдвиги** | **Кельвин (K):** $C = K - 273.15$<br>**Фаренгейт (F):** $C = (F - 32) \times 5/9$<br>**Реомюр (Re):** $C = 0.8 \times Re$ |
| **BR-03** | **Интерполяция твердости** | Используется таблица `HARDNESS_DATA` (см. Раздел 6).<br>Логика: поиск интервала $[X_i, X_{i+1}]$ и линейная интерполяция значения. Если значение выходит за границы таблицы — возвращается `0.0` (экстраполяция запрещена). |
| **BR-04** | **Коды и UI** | При поиске единицы (`from_u`, `to_u`) поиск ведется строго по полю **«Единица (код)»**. Поле **«Единица (UI)»** используется только для отображения во фронтенде/отчетах. |

**Раздел 7. Совместимость с внешними библиотеками (Integration)**

Библиотека `UniConv+` должна служить слоем абстракции (Adapter) между пользовательским вводом и расчетными ядрами.
Основной сценарий интеграции — подготовка данных для библиотеки свойств воды и пара `seuif97` (IAPWS-IF97).

Для обеспечения совместимости реализовать метод или справочник `get_target_si_unit(type_en)`, возвращающий единицу измерения, которую ожидает `seuif97`.

**Таблица маппинга (Target SI):**

| Тип величины (En) | Target Unit (Код) | Комментарий |
| :--- | :---: | :--- |
| `pressure` | **МПа** | IAPWS использует МПа, а не Па |
| `temperature` | **C** | Основной ввод в C (функции `pt`, `ph`) |
| `enthalpy` | **кДж/кг** | |
| `entropy` | **кДж/(кг*К)** | |
| `specific_volume` | **м3/кг** | |
| `density` | **кг/м3** | |
| `dyn_viscosity` | **Па*с** | |
| `kin_viscosity` | **м2/с** | |
| `thermal_conductivity` | **Вт/(м*К)** | |
| `dryness` | **-** | Безразмерная (0..1) |
| `dimensionless` | **-** | |
| `efficiency` (КПД) | **-** | Доли единицы (не %) |

*Пример использования:*
```python
# Логика клиента
user_p = 100 # бар
target_unit = uc.get_target_unit("pressure") # вернет "МПа"
p_for_if97 = uc.convert(user_p, "бар", target_unit, "pressure")
# Вызов if97(p_for_if97, ...)
```

**Раздел 8. Критерии приемки (Definition of Done)**

Задача считается выполненной, если реализован класс `UnitConverter`, удовлетворяющий следующим требованиям:

1.  **Двуязычный API:**
    *   Методы принимают `parameter_type` как на русском (напр. `"Давление"`), так и на английском (напр. `"pressure"`).
    *   Реализована нормализация строк (игнорирование регистра и пробелов).

2.  **Структура данных:**
    *   Метод `get_available_units` возвращает полную структуру для UI: код, UI-название, полное название (если есть).
    *   Все коэффициенты из **Раздела 5** перенесены в код с точностью, соответствующей спецификации (без округлений).

3.  **Логика конвертации:**
    *   Реализована сквозная конвертация `convert(val, from, to, type)`.
    *   Работает конвертация температуры: C ↔ K ↔ F ↔ Re (Реомюр).
    *   Работает интерполяция твердости:
        *   Точное совпадение значения из таблицы.
        *   Линейная интерполяция между точками.
        *   Возврат `0.0` (или `None`) при выходе за границы диапазона таблицы.

4.  **Качество кода:**
    *   Код покрыт Unit-тестами (минимум 1 тест на каждый физический тип величины + тесты на граничные условия твердости).
    *   Отсутствуют "магические числа" в формулах (все коэффициенты вынесены в константы или конфиг).
    *   Погрешность обратной конвертации (`A -> B -> A`) не превышает `1e-5` (за исключением табличных значений твердости, где возможны потери точности интерполяции).

5.  **Артефакты:**
    *   Исходный код библиотеки (`.py`).
    *   Набор тестов (`tests/`).
    *   Файл `README.md` с примерами использования (Ru/En имена, Hardness, UI-units).
